# مطبعة شمس للدعاية والإعلان — نظام إدارة الفواتير والحسابات

برنامج سطح مكتب (Windows) لإدارة حسابات وفواتير المطابع، مبني بـ **Flutter** + **SQLite**.

---

## ✨ المزايا المنفذة

- ✅ إدارة العملاء (إضافة / تعديل / حذف / بحث)
- ✅ إدارة المواد والخدمات مع **سعرين** (مفرد وجملة) لكل مادة
- ✅ دعم وحدات القياس: عدد، متر مربع، متر، سم، ملم، ورقة، رول
- ✅ حساب **المساحة والسعر تلقائياً** عند إدخال الطول والعرض والكمية
- ✅ إنشاء فواتير باختيار نوع الحساب (مفرد / جملة) مع تعديل السعر يدوياً **بدون التأثير على السعر الأساسي المحفوظ**
- ✅ خصم بالنسبة أو بالقيمة + ملاحظات
- ✅ ترقيم فواتير تلقائي (Prefix قابل للتخصيص)
- ✅ تصدير الفاتورة PDF بجودة عالية (بتصميم احترافي ثنائي اللون يدعم العربية بالكامل RTL)
- ✅ طباعة مباشرة من داخل البرنامج
- ✅ زر لإرسال الفاتورة عبر واتساب (يحفظ PDF ويفتح واتساب)
- ✅ رفع شعار المطبعة وتخصيص معلومات التواصل من الإعدادات
- ✅ تخصيص اللون الرئيسي للتطبيق والفواتير
- ✅ الوضع الليلي / النهاري
- ✅ أرشفة كاملة للفواتير + بحث + فلترة حسب الحالة (معلقة / مدفوعة / جزئي / ملغاة)
- ✅ لوحة رئيسية (Dashboard) بإحصائيات سريعة

> **ملاحظة:** خانة QR Code وقوالب الفاتورة المتعددة (قالب 2 و3) جاهزة في الإعدادات كخيارات، لكن التصميم الفعلي المطبّق حالياً في `pdf_generator.dart` هو القالب الأول (الكلاسيكي الاحترافي). يمكن توسيعه لاحقاً بسهولة لأن التصميم منظم في دوال واضحة.

## 🛠️ التشغيل لأول مرة (على جهاز Windows)

### 1. المتطلبات
- تثبيت [Flutter SDK](https://docs.flutter.dev/get-started/install/windows) (مع دعم Windows desktop)
- تفعيل دعم سطح المكتب:
  ```
  flutter config --enable-windows-desktop
  ```
- Visual Studio 2022 مع حزمة **"Desktop development with C++"**

### 2. إضافة خط Cairo (مطلوب لدعم العربية في PDF والواجهة)
نزّل خط **Cairo** (مجاني من Google Fonts):
https://fonts.google.com/specimen/Cairo

ضع الملفات التالية داخل `assets/fonts/`:
```
Cairo-Regular.ttf
Cairo-Medium.ttf
Cairo-SemiBold.ttf
Cairo-Bold.ttf
```

> إن لم تُضف الملفات، التطبيق سيعمل لكن سيستخدم خطاً افتراضياً قد لا يدعم العربية بشكل مثالي في PDF.

### 3. تثبيت الحزم
داخل مجلد المشروع:
```
flutter pub get
```

### 4. توليد ملفات Windows (مرة واحدة فقط)
بعض ملفات نظام Windows (`windows/` الكاملة بإعدادات CMake) تُنشأ تلقائياً من Flutter ولم تُدرج هنا. شغّل:
```
flutter create --platforms=windows .
```
هذا الأمر **لن يحذف الكود الموجود**، فقط يضيف/يحدّث ملفات تكوين Windows الأصلية (CMakeLists, runner, إلخ).

### 5. التشغيل
```
flutter run -d windows
```

### 6. بناء نسخة نهائية (exe) للتوزيع
```
flutter build windows
```
الملف التنفيذي سيكون في:
```
build/windows/x64/runner/Release/
```

## 🧩 ملاحظات تقنية مهمة

1. **سعر العنصر الأساسي محفوظ في جدول `items`** (مفرد/جملة)، وعند إضافته للفاتورة يُنسخ السعر إلى `invoice_items.unit_price` — فتعديله داخل الفاتورة لا يغيّر السعر الأساسي أبداً.
2. **حساب المساحة:** يتم تلقائياً عند اختيار وحدة (مم/سم/م/م²) وإدخال الطول والعرض، وفق الصيغة:
   `الإجمالي = المساحة (م²) × الكمية × سعر الوحدة`
   أما لوحدة "عدد" فالحساب: `الإجمالي = الكمية × سعر الوحدة` مباشرة.
3. **واتساب:** الزر يحفظ الفاتورة PDF محلياً ثم يفتح واتساب ويب لإرسالها يدوياً (واتساب لا يسمح بإرفاق ملفات تلقائياً عبر الروابط لأسباب أمنية).
4. **QR Code:** مكتبة `qr_flutter` مضافة في `pubspec.yaml`؛ لتفعيلها في الفاتورة، يمكن إضافة ويدجت `QrImageView` داخل دالة `_buildHeader` في `pdf_generator.dart` (مثلاً QR يحتوي رقم الفاتورة + الإجمالي).
